# Genetic Algorithm Performance Analysis

## Overview
This project explores the performance of a **Genetic Algorithm (GA)** under different configurations of **crossover rates** and **mutation rates**.  
The goal is to analyze how these parameters affect **convergence speed** and **final fitness values** over multiple generations.

## Features
- Configurable parameters in `GAConfig.java`, including:  
  - Custom random seeds  
  - Input data file  
  - Crossover rate  
  - Mutation rate  
- Supports multiple **crossover types**:  
  - One-point crossover  
  - Uniform order crossover  
- Outputs results for analysis:  
  - Console output of **best fitness**, **best key found**, and **attempted decryption**  
  - `output.csv` containing **best fitness** and **average fitness** per generation  

## Running the Program
1. Set all parameters in `GAConfig.java` according to your needs.  
2. Run `main.java`.  
3. View results in the console and check the generated `output.csv`.  

## Input Data Format
- Input files (`data1.txt`, `data2.txt`, etc.) must follow this structure:  
  1. **First line**: Maximum key length (integer)  
  2. **Remaining lines**: Encrypted text  

- Example keys:  
  - `data1.txt` key: `"thisisasupersecurepassword"`  
  - `data2.txt` key: `"supercalifragilisticexpialidocious"`  

- Alternatively, encrypted text and max key length (ChromLen) can be specified directly in `GAConfig.java`.  

## Results
- The `results` folder contains **graphs** showing:  
  - Average and best fitness per generation  
  - Comparisons of **different crossover and mutation parameters**  
  - Both **data files** and **crossover types** are represented  

- These graphs are generated by running the program multiple times and combining the data.
